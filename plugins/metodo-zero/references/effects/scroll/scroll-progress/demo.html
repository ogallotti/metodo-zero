Scroll Progress

A sleek progress bar fixed at the top of the page that tracks your scroll position. Scroll down to see it in action.

Why scroll indicators matter

Scroll progress indicators give users a clear sense of how much content remains. They reduce anxiety on long pages by providing spatial context, turning an unknown journey into a measurable one.

This implementation uses a lightweight requestAnimationFrame loop to track scroll position with minimal performance impact. The bar width maps directly to the percentage of the page scrolled.

All visual properties are exposed through CSS custom properties, making it trivial to match your brand. Adjust the height, color, and z-index without touching any JavaScript.

Technical highlights

The progress calculation accounts for viewport height, ensuring the bar reaches 100% exactly when you hit the bottom of the page. Passive event listeners keep scrolling buttery smooth.

Accessibility is built in: the progress track carries proper ARIA attributes including role="progressbar" and dynamically updated aria-valuenow. Screen readers can announce scroll progress when needed.

The reduced-motion media query is respected — when the user prefers reduced motion, the transition on the bar is removed entirely, ensuring an instant visual update with no animation.

Integration notes

Drop the script into any page and it creates the DOM elements automatically. No initialization required. The bar inserts itself as the first child of the body element.

For single-page applications with client-side routing, you may want to reset the bar on navigation. Simply dispatch a scroll event or call the update function after route changes.

The z-index is set high by default (9999) to ensure the bar appears above most content. If your site has modal overlays or sticky headers at higher z-index values, adjust the --progress-z variable accordingly.

Customization options

Change the bar color with --progress-color. Gradient backgrounds work too — just set the CSS variable to a gradient value or override the background property directly on .scroll-progress-bar.

The track background defaults to transparent, meaning the bar floats invisibly until it fills. Set --progress-bg to a subtle color for a visible track effect, similar to a traditional progress bar.

Height ranges from a barely-visible 2px hairline to a bold 12px strip. The sweet spot for most designs sits around 3-4px — noticeable without being distracting.
